<task type="auto">
  <name>Task 2: Install flyctl, set secrets, and deploy to Fly.io (DEPL-02)</name>
  <files>fly.toml</files>
  <action>
    Step 1: Read FLY_API_TOKEN and DEEPGRAM_API_KEY from /coding/deepgram-secretary/.env:
    ```bash
    grep -E "FLY_API_TOKEN|DEEPGRAM_API_KEY" /coding/deepgram-secretary/.env
    ```

    Step 2: Install flyctl (Linux):
    ```bash
    curl -L https://fly.io/install.sh | sh
    export FLYCTL_INSTALL="$HOME/.fly"
    export PATH="$FLYCTL_INSTALL/bin:$PATH"
    ```
    Verify: `fly version`

    Step 3: Authenticate non-interactively:
    ```bash
    export FLY_API_TOKEN="<value from .env>"
    ```

    Step 4: Check if DEEPGRAM_API_KEY secret already exists on Fly:
    ```bash
    fly secrets list --app deepgram-python-stt
    ```
    If DEEPGRAM_API_KEY is listed: skip Step 5 (already set).
    If DEEPGRAM_API_KEY is NOT listed: run Step 5.

    Step 5 (conditional): Stage the secret:
    ```bash
    fly secrets set DEEPGRAM_API_KEY="<value from .env>" --stage --app deepgram-python-stt
    ```
    Use --stage so it takes effect during the upcoming deploy, not immediately (avoids triggering a separate machine restart).

    Step 6: Deploy:
    ```bash
    cd /coding/deepgram-python-stt && fly deploy --app deepgram-python-stt
    ```
    Expected: Build completes, machine starts, health check passes.

    Step 7: Verify deployment status:
    ```bash
    fly status --app deepgram-python-stt
    curl -s https://deepgram-python-stt.fly.dev/ | head -5
    ```
    Expected: fly status shows machine running; curl returns HTML (200).
  </action>
  <verify>

  </verify>
