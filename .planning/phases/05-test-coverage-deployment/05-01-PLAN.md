From tests/conftest.py:
- `sio_client` is a session-scoped `socketio.AsyncClient` fixture already connected to the test server
- Use `asyncio.get_running_loop().create_future()` + `sio_client.on(event)` pattern for events that emit a response
- Always guard with `if not future.done():` before `future.set_result(data)` (prevents duplicate-listener pollution across session)
- `await asyncio.wait_for(future, timeout=5.0)` to await the response

From tests/test_app.py (existing pattern to follow):
```python
async def test_start_file_streaming_no_filename_emits_error(sio_client):
    future = asyncio.get_running_loop().create_future()

    @sio_client.on("stream_error")
    def on_error(data):
        if not future.done():
            future.set_result(data)

    await sio_client.emit("start_file_streaming", {})
    result = await asyncio.wait_for(future, timeout=5.0)
    assert "message" in result
```
